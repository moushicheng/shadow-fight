# 霜蚀机制重做：从永久控制改为蓄力-爆发循环

## 做什么

将霜蚀从"无上限叠加 + 自然衰减"的永久控制模式，改为"蓄力减速 → 触发冻结 → 霜蚀转伤害 → 清零重来"的爆发循环模式。

## 为什么

当前机制下，冰系可以轻松实现永久冻结（特别是配合冰封领域/永冬），对手完全无法行动，体验极差。新机制让冻结变成有限窗口的爆发时机，同时保留霜蚀减速的策略价值。

## 核心规则变更

1. **霜蚀不再自然衰减**（移除 frostDecayPerCycle）
2. **冻结触发时清零霜蚀**（effective_speed ≤ 0 → 消耗全部霜蚀，进入冻结状态）
3. **冻结持续到下一周期开始**（周期结算时解冻）
4. **冻结期间霜蚀转伤害**（1:1 转化，走护甲结算）
5. **解冻时行动槽归零**

## 涉及范围

- 类型：CharacterTypes（新增 frozenUntilCycle 字段）、BattleTypes（移除/调整 frostDecayPerCycle）
- 状态管理：StatusManager（冻结触发、霜蚀转伤害、解冻逻辑）
- 战斗引擎：BattleEngine（冻结判定改用显式状态而非实时计算）
- 周期结算：CycleResolver（周期开始时解冻）
- 效果解析：CardEffectResolver（冻结期间霜蚀转伤害）
- 有效速度：EffectiveStats（冻结状态判定调整）
- 卡牌数据：ice.ts（永冬重设计、卡牌描述更新）
- 设计文档：battle-base.md、ice/cards.md

## 约束和边界

- 转化比例 1:1（每层霜蚀 = 1 伤害），走护甲
- 冻结时间固定为"到下一周期开始"，不受霜蚀层数影响
