# UI 与交互设计

本文档定义各场景的 UI 布局与交互方式。目标平台为抖音小游戏，**竖屏**，触控操作。

---

## 一、全局规范

| 项目 | 规范 |
| --- | --- |
| 屏幕方向 | 竖屏（Portrait） |
| 安全区 | 顶部留出状态栏 + 刘海区域，底部留出手势条区域 |
| 字号下限 | 正文 ≥ 28px（确保小屏可读） |
| 触控热区 | 所有可点击元素 ≥ 44×44pt |
| 动效原则 | 轻量为主，过渡动画 ≤ 300ms，避免阻塞操作 |

---

## 二、战斗界面

战斗完全自动，玩家只观战，不做卡牌操作。双方交替出牌，需同时展示敌我双方的出牌信息。

```
┌─────────────────────────┐
│  👤对手  HP [██████░░░░] │  ← 对手头像、昵称、HP 条
│  MP [████░░░░░░]         │  ← 对手 MP 条
│  霜蚀×3  灼烧×5  毒×2   │  ← 对手身上的状态效果
│  ┌───────────────────┐  │
│  │  对手当前出牌       │  │  ← 对手打出的卡牌（翻面展示名称+效果）
│  └───────────────────┘  │
│  ─────────────────────  │
│  ┌───────────────────┐  │
│  │                   │  │
│  │    战斗动画区      │  │  ← 角色立绘 / 简易动画 / 伤害数字
│  │                   │  │
│  └───────────────────┘  │
│  ─────────────────────  │
│  ┌───────────────────┐  │
│  │  玩家当前出牌       │  │  ← 玩家打出的卡牌（翻面展示名称+效果）
│  └───────────────────┘  │
│  ┌──┐ ┌──┐ ┌──┐ ┌──┐   │
│  │①│ │②│ │③│ │④│   │  ← 玩家接下来 4 张缩略图
│  └──┘ └──┘ └──┘ └──┘   │
│  霜蚀×0  灼烧×0  毒×0   │  ← 玩家身上的状态效果
│  👤玩家  HP [████████░░] │  ← 玩家 HP 条
│  MP [██████░░░░]  ⚙️×5  │  ← MP 条 + 遗物图标
│     [加速]   [跳过]     │  ← 操作按钮
└─────────────────────────┘
```

**信息不对称设计**：玩家侧显示接下来 4 张卡缩略图（因为是自己构筑的卡组，看到预览有"验证策略"的满足感），对手侧不显示（玩家无法对此做出反应，展示只会增加视觉噪音）。

### 2.1 行动时间轴

在战斗动画区上方或下方显示双方行动槽进度条：

```
玩家 ████████░░ 80%     对手 ██████░░░░ 60%
```

行动槽满时触发出牌动画，让玩家直观感知"谁更快"。

### 2.2 速度变化反馈

霜蚀 / 汲取导致速度变化时，在角色状态区显示变化动画（如 `SPD 10 → 7 ↓`），同时行动槽填充速率可见地变化。

### 2.3 战斗日志

侧边可展开的折叠面板，记录每次行动（默认收起，点击展开）：

```
[Tick 10] 玩家「火焰冲击」→ 16 伤害，灼烧+2
[Tick 15] 对手「铁壁」→ 获得 15 护甲
[Cycle 1] 周期结算：毒药伤害 5，霜蚀 12→10，MP+1
```

## 三、卡组管理界面

在商店"调整卡序"和卡组浏览时使用。

### 3.1 网格布局

卡组以**每行 4 张**的网格排列，序号从左到右、从上到下递增，代表出牌顺序：

```
┌─────────────────────────┐
│  ◀ 卡组编辑 (12张)       │
│                          │
│  ┌──┐ ┌──┐ ┌──┐ ┌──┐   │
│  │①│ │②│ │③│ │④│   │
│  │🔥│ │❄️│ │⚔️│ │🔥│   │   ← 流派色 + 缩略图标
│  │ 2│ │ 1│ │ 1│ │ 2│   │   ← 法力费用
│  └──┘ └──┘ └──┘ └──┘   │
│  ┌──┐ ┌──┐ ┌──┐ ┌──┐   │
│  │⑤│ │⑥│ │⑦│ │⑧│   │
│  │🛡️│ │💀│ │🔥│ │❄️│   │
│  │ 3│ │ 0│ │ 2│ │ 1│   │
│  └──┘ └──┘ └──┘ └──┘   │
│  ┌──┐ ┌──┐ ┌──┐ ┌──┐   │
│  │⑨│ │⑩│ │⑪│ │⑫│   │
│  │⚔️│ │🔥│ │❄️│ │💀│   │
│  │ 1│ │ 2│ │ 3│ │ 0│   │
│  └──┘ └──┘ └──┘ └──┘   │
│                          │
│  点击查看 · 长按拖动调序  │
└─────────────────────────┘
```

### 3.2 单张卡面信息

每张缩略卡面包含三层信息：

| 层级 | 内容 | 说明 |
| --- | --- | --- |
| 顶部 | 序号（①②③...） | 出牌顺序，白色加粗 |
| 中部 | 流派图标 | 带流派底色（火红/冰蓝/毒绿/通用灰等） |
| 底部 | 法力费用 | 数字，便于规划费用节奏 |

### 3.3 交互操作

| 操作 | 行为 | 说明 |
| --- | --- | --- |
| 点击 | 弹出卡牌详情浮层 | 展示完整卡面：名称、效果文本、品质、标签、升级状态 |
| 长按 | 进入拖动模式 | 卡牌浮起跟随手指，其余卡牌实时让位，序号实时重排 |
| 松手 | 落位确认 | 卡牌放入新位置，所有序号更新，带轻微弹性动效 |

### 3.4 调序反馈

拖动过程中的关键反馈：

- 被拖动的卡牌**放大 110%** + 半透明阴影，区别于静态卡牌
- 原位置显示**虚线占位框**
- 其他卡牌实时平滑移动让位，序号随之变化
- 松手时**轻震动反馈**（如设备支持）

### 3.5 容量参考

| 卡组张数 | 行数 | 是否需要滚动 |
| --- | --- | --- |
| 8 张 | 2 行 | 否 |
| 12 张 | 3 行 | 否 |
| 16 张 | 4 行 | 否 |
| 20 张 | 5 行 | 极少量滚动 |

> 大多数对局卡组在 8-16 张范围内，一屏可完整展示。

## 九、交互设计原则

| 原则 | 说明 |
| --- | --- |
| 单手可操作 | 所有核心操作在拇指热区内（屏幕下半部分），信息展示在上半部分 |
| 最多两步到达 | 任何操作最多点击 2 次完成（如：点击卡牌 → 确认购买） |
| 可撤销 | 卡组调序支持撤销；商店购买前有确认弹窗 |
| 信息分层 | 首屏展示关键信息（图标+数字），详情按需展开（点击/长按） |
| 零教学可理解 | 使用通用手机交互范式（滚动、点击、长按拖动），不发明新手势 |
