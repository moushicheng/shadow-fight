# 战斗系统设计

## 一、设计理念

### 1.1 完全自动战斗

战斗中玩家**不做任何操作**，系统按卡组顺序自动出牌。玩家的所有策略决策发生在战斗之外（卡组构筑、卡序调整、商店购买、事件选择）。

UI 提供「加速」和「跳过」按钮，方便快速看结果。

### 1.2 核心原则

| 原则 | 说明 |
| ---------- | ---------------------------------------------------------------- |
| 确定性 | 相同输入 = 相同结果（便于回放和验证） |
| 速度平滑 | 速度差异对行动频率的影响是**连续的**，不存在"差 1 速无区别、差 2 速天差地别"的阶梯感 |
| 可读性 | 观战时玩家能直观理解"谁更快、快多少" |
| MP 公平 | 高速角色获得更多行动机会，但不额外获得更多法力——MP 按固定时间恢复 |

## 二、行动时间轴（ATB 系统）

### 2.1 原方案问题

原公式 `floor(己方速度 / min(双方速度))` 存在三个核心问题：

| 问题 | 示例 |
| ---------- | -------------------------------------------------------------------- |
| 阶梯效应 | SPD 6 vs 5 → 双方都是 1 次行动，速度优势完全浪费；SPD 10 vs 5 → 2:1，突然翻倍 |
| 不直觉 | 玩家难以预判"再加几点速度才能多一次行动" |
| 动态速度难处理 | 霜蚀 / 汲取改变速度后需要实时重算 `min(双方速度)`，行动次数容易突变 |

### 2.2 ATB 机制

采用 **Action Time Battle（行动时间轴）** 系统，行动频率与速度**严格成正比**。

**核心规则**：

1. 双方各有一个 **行动槽（Action Gauge）**，初始值为 0
2. 战斗以 **tick** 为最小时间单位循环推进
3. 每个 tick：双方行动槽各增加自身当前 **有效速度（effective_speed）**
4. 当行动槽 **≥ 100** 时，该角色执行一次行动（打出卡组中下一张牌），行动槽 **-= 100**（保留溢出值）
5. 若同一 tick 双方行动槽都 ≥ 100：**有效速度高者先行动**；速度相同则**攻方（玩家）优先**

**伪代码**：

```
gaugeA = 0, gaugeB = 0

while battle_not_over:
    tick_count += 1
    gaugeA += effectiveSpeed(A)
    gaugeB += effectiveSpeed(B)

    actors = []
    if gaugeA >= 100: actors.append(A)
    if gaugeB >= 100: actors.append(B)

    sort actors by effectiveSpeed desc (tie → attacker first)

    for actor in actors:
        playNextCard(actor)
        actor.gauge -= 100
        checkBattleEnd()

    if tick_count % 100 == 0:
        cycleEnd()  // 周期结算（状态衰减、毒伤、MP回复等）
```

### 2.3 速度与行动频率的关系

| A 速度 | B 速度 | 行动频率比（A : B） | 效果描述 |
| ------ | ------ | ------------------- | ---------------------------------------- |
| 10 | 10 | 1 : 1 | 完全同步，交替行动 |
| 12 | 10 | 6 : 5 | A 稍快，每 5 次对方行动中 A 多打 1 张牌 |
| 15 | 10 | 3 : 2 | A 明显快 |
| 20 | 10 | 2 : 1 | A 打 2 张，B 才打 1 张 |
| **6** | **5** | **6 : 5** | **微小优势也能体现**（原方案下为 1:1） |

> **关键优势**：在 ATB 系统下，SPD 6 vs 5 的差异也能被精确反映，不会像原方案那样被 `floor` 吃掉。行动频率比恒等于速度比——直觉、平滑、可预测。

### 2.4 行动顺序示例

以 SPD 10 vs SPD 7 为例（100 ticks 内）：

```
Tick 10: A 行动（第1张牌）
Tick 15: B 行动（第1张牌）
Tick 20: A 行动（第2张牌）
Tick 29: B 行动（第2张牌）
Tick 30: A 行动（第3张牌）
Tick 40: A 行动（第4张牌）
Tick 43: B 行动（第3张牌）
Tick 50: A 行动（第5张牌）
Tick 57: B 行动（第4张牌）
Tick 60: A 行动（第6张牌）
Tick 70: A 行动（第7张牌）
Tick 71: B 行动（第5张牌）
Tick 80: A 行动（第8张牌）
Tick 86: B 行动（第6张牌）
Tick 90: A 行动（第9张牌）
Tick 100: A 行动（第10张牌），B 行动（第7张牌）← 同 tick，A 速度高先手
         → 周期结算
```

**结果**：100 ticks 内 A 行动 10 次、B 行动 7 次，精确等于速度比 10:7。

### 2.5 有效速度

```
effective_speed = base_speed + speed_buffs - floor(霜蚀层数 / 3) - 汲取损失
```

| 分量 | 说明 |
| ---------- | ------------------------------------------------ |
| base_speed | 角色基础 SPD（含事件 / 遗物的永久修改） |
| speed_buffs | 战斗中临时加速效果（刺客加速牌等） |
| 霜蚀减速 | 每 **3 层**霜蚀 = -1 有效速度 |
| 汲取损失 | 被血族汲取的 SPD 值 |

**有效速度下限 = 0**：当 effective_speed ≤ 0 时进入**冻结状态**。

### 2.6 冻结状态

| 项目 | 规则 |
| -------- | ------------------------------------------------------ |
| 触发条件 | effective_speed ≤ 0 |
| 效果 | 行动槽停止增长，角色无法行动 |
| 周期结算 | 仍然正常进行（霜蚀衰减、毒药伤害等照常） |
| 解冻 | 霜蚀每周期 -2 层，速度恢复 > 0 后自动解冻，行动槽从 0 继续累积 |

## 三、周期（Cycle）与状态结算

### 3.1 周期定义

- **1 周期 = 100 ticks**
- 每满 100 ticks 触发一次**周期结算（Cycle End）**

### 3.2 周期结算顺序

| 顺序 | 结算内容 | 说明 |
| ---- | -------- | ---------------------------------------------------- |
| 1 | 毒药伤害 | 双方受到 = 各自毒药层数的伤害（无视护甲） |
| 2 | 状态衰减 | 霜蚀：每周期 -2 层；毒药：每周期 -1 层（先伤害后衰减） |
| 3 | 法力回复 | 双方各回复 **1 MP**（不超上限） |
| 4 | 加时伤害 | 若已进入加时阶段，双方受递增伤害（见 §八） |
| 5 | 周期计数 | cycle_count += 1 |

### 3.3 法力（MP）系统

| 规则 | 说明 |
| ---------- | ---------------------------------------------------- |
| 初始 MP | 战斗开始时 = 最大 MP = `round(MANA × 1.5)` |
| 周期回复 | 每周期 +1 MP（不超上限） |
| 消耗 | 打出卡牌时扣除该卡费用 |
| MP 不足 | 该卡牌被**跳过**（不打出、不消耗 MP），卡组指针仍前进 |
| 跳过代价 | 本次行动浪费（空过），行动槽正常消耗 |

> **设计意图**：MP 按时间恢复而非按行动次数恢复，确保高速角色不会同时获得"更多出牌 + 更多法力"的双重优势。速度带来行动频率，MP 是独立的资源维度。高速低蓝角色需要依赖低费卡，高蓝低速角色可以安排大招——两种构筑方向各有价值。

## 四、战斗初始化

### 4.1 战斗开始流程

```
1. 加载双方数据（属性、卡组、遗物）
2. 应用战斗开始效果：
   a. 遗物被动触发（如"铁皮水壶：获得 5 护甲"）
   b. 临时 Buff 生效（来自事件 / 赌约的本场增益）
3. 清零战斗内状态：
   - 护甲 = 0（遗物加成在此之后叠加）
   - 霜蚀 / 灼烧 / 毒药 = 0
   - 诅咒卡从卡组中移除
   - 汲取效果清零（被偷属性恢复）
4. 初始化时间轴：
   - 双方 action_gauge = 0
   - tick_count = 0, cycle_count = 0
5. 初始化卡组指针：双方 deck_index = 0
6. 战斗开始
```

### 4.2 遗物触发顺序

先触发**主场方（玩家）**，再触发**客场方（残影 / 野怪）**。同一方内按遗物获取顺序依次触发。

## 五、行动解析

### 5.1 单次行动流程

当角色的行动槽 ≥ 100 时，执行以下流程：

```
1. 行动槽 -= 100（保留溢出值）
2. 读取卡组中 deck_index 位置的卡牌
3. 检查 MP ≥ 卡牌费用：
   a. 足够 → 扣除 MP，执行卡牌效果
   b. 不足 → 跳过此卡（不消耗 MP，不触发效果）
4. deck_index += 1
5. 若 deck_index ≥ 卡组长度 → deck_index = 0（循环回第一张）
6. 检查战斗结束条件
```

### 5.2 卡牌效果类型

| 效果类型 | 说明 |
| -------- | ------------------------------------------ |
| 造成伤害 | 见 §六 伤害计算 |
| 施加状态 | 给目标添加状态层数（霜蚀 / 灼烧 / 毒药） |
| 获得护甲 | 增加自身护甲值 |
| 回复 HP | 恢复生命值（不超过 maxHp） |
| 回复 MP | 恢复法力值（不超过上限） |
| 汲取属性 | 从对方转移 ATK / SPD / 护甲到自身 |
| 插入诅咒 | 将诅咒卡插入对方卡组 |
| 自伤 | 对自身造成伤害（狂战士 / 诅咒卡） |
| 条件效果 | 根据条件执行不同分支（如"若 HP < 30% 则…"） |

卡牌包含多个效果时，按文本顺序依次结算。

### 5.3 诅咒卡处理

**插入规则**：

- 诅咒卡被插入对方卡组的**当前指针之后、卡组末尾之前**的随机位置
- 确保不会被立即跳过，也不会要等整圈卡组才遇到

**打出规则**：

- 当轮到诅咒卡时，**强制执行**其效果（自伤 / 浪费 MP 等）
- 打出后从卡组中**永久移除**
- 后续卡牌自动前移，卡组长度 -1

## 六、伤害计算

### 6.1 基础伤害

```
基础伤害 = 卡牌效果中的伤害表达式求值
```

卡牌伤害表达式可引用战斗属性：

| 表达式示例 | 含义 |
| --------------- | ---------------------- |
| `0.5 × ATK` | 普通攻击 |
| `1 × SPD` | 刺客轻击（速度缩放） |
| `已损失HP × 0.3` | 狂战怒击（残血缩放） |
| `当前护甲` | 守卫甲转伤害 |

### 6.2 伤害修正

```
最终伤害 = floor(基础伤害 × 伤害倍率 + 灼烧加成)
```

| 修正项 | 计算 |
| -------- | -------------------------------------------------- |
| 伤害倍率 | 1.0 + 各种百分比加成之和（赌约、遗物等） |
| 灼烧加成 | 仅限**火系卡牌**：+ 目标当前灼烧层数 |

### 6.3 护甲结算

```
if 目标护甲 > 0:
    护甲消耗 = min(最终伤害, 目标护甲)
    目标护甲 -= 护甲消耗
    实际扣血 = 最终伤害 - 护甲消耗
else:
    实际扣血 = 最终伤害

目标HP -= 实际扣血
```

### 6.4 特殊伤害规则

| 类型 | 规则 |
| ---------- | ---------------------------------------- |
| 毒药伤害 | **无视护甲**，直接扣 HP |
| 灼烧引爆 | 消耗全部灼烧层数，造成 `层数 × 引爆倍率` 伤害，走正常护甲结算 |
| 自伤 | **无视自身护甲**，直接扣 HP |
| 吸血 | 造成伤害后回复等量（或按比例）HP，回复量基于实际造成的伤害（扣甲后） |

## 七、状态效果详解

### 7.1 关键词状态

#### 霜蚀（冰系）

| 属性 | 值 |
| -------- | ------------------------------------------------------------ |
| 来源 | 冰系卡牌 |
| 效果 | 降低目标有效速度：每 **3 层**霜蚀 = effective_speed **-1** |
| 冻结 | effective_speed ≤ 0 时触发，目标无法行动 |
| 衰减 | 每周期 -2 层 |
| 上限 | 无硬上限（但速度到 0 后继续叠只延长冻结时间） |

**示例**：目标 SPD 8，被施加 24 层霜蚀：

```
effective_speed = 8 - floor(24/3) = 8 - 8 = 0 → 冻结！
周期1衰减: 24→22层, speed = 8-7 = 1 → 解冻
周期2衰减: 22→20层, speed = 8-6 = 2
...逐步恢复
```

**UI 展示原则**（降低玩家认知负担，不要求玩家自行计算 `floor(层数/3)`）：

| 场景 | 展示方式 | 示例 |
| --- | --- | --- |
| 战斗中状态图标 | 同时显示**层数**和**减速值** | `霜蚀 24 · 减速 8` |
| 冻结状态 | 醒目提示冻结 + 预估解冻周期数 | `冻结中！约 1 周期后解冻` |
| 卡牌效果文本 | 在层数后括号注明换算减速值 | `施加 6 层霜蚀（减速 2）` |
| 速度变化反馈 | 直接显示速度变化结果 | `SPD 8 → 0 ↓ 冻结！` |

#### 灼烧（火系）

| 属性 | 值 |
| -------- | -------------------------------------------------- |
| 来源 | 火系卡牌 |
| 效果 | 不直接造成伤害；**火系卡牌**对目标伤害 + 灼烧层数 |
| 引爆 | 部分卡牌可消耗全部灼烧层数，造成 `层数 × 引爆倍率` 伤害 |
| 衰减 | **不自然衰减** |
| 清零 | 引爆后清零 |

**节奏**：前几张牌叠灼烧层数 → 中期火系普通牌享受加成 → 引爆卡一波收割。卡序编排是火系的核心策略。

#### 毒药（毒系）

| 属性 | 值 |
| -------- | ------------------------------------------ |
| 来源 | 毒系卡牌 |
| 效果 | 每周期结算时造成 = 层数的伤害，**无视护甲** |
| 衰减 | 每周期 -1 层（先造成伤害，再衰减） |

#### 诅咒（咒术师）

| 属性 | 值 |
| -------- | ---------------------------------------------------- |
| 来源 | 咒术师卡牌 |
| 效果 | 诅咒卡被塞入对手卡组，轮到时强制使用 |
| 移除 | 诅咒卡被打出后从卡组永久移除 |
| 卡组影响 | 增加卡组总长度，稀释正常牌出现频率，打乱出牌节奏 |

**诅咒卡类型示例**（完整列表见后续设计）：

| 诅咒卡 | 费用 | 效果 |
| ------ | ---- | -------------------------------- |
| 虚弱 | 2 | 无效果（纯浪费 MP 和行动机会） |
| 噬魂 | 0 | 自伤 10 HP |
| 迷雾 | 3 | 无效果（高费浪费） |

#### 汲取（血族）

| 属性 | 值 |
| ---------- | -------------------------------------------------- |
| 来源 | 血族卡牌 |
| 可汲取属性 | ATK、SPD、护甲 |
| 效果 | 从对方扣除 N 点并加到自身（实际差值变化 = 2N） |
| 持续时间 | 本场战斗 |
| SPD 汲取 | **立即影响**双方行动槽填充速率——血族对抗速度流的关键手段 |

**ATB 联动**：汲取 SPD 后，对方行动槽填充变慢、己方变快，后续行动顺序实时调整。这比原方案（重算 floor 公式）自然得多。

### 7.2 通用状态

#### 护甲

| 属性 | 值 |
| ------ | -------------------------------------------- |
| 来源 | 多流派卡牌 / 遗物 |
| 效果 | 受击时先扣护甲再扣 HP |
| 初始值 | 每场战斗 = 0（遗物可在战斗开始时叠加） |
| 上限 | 无上限 |

## 八、战斗结束条件

### 8.1 正常结束

任一方 HP ≤ 0 → 该方**死亡**，另一方胜利。

若某次结算导致**双方同时 HP ≤ 0**（如毒药同 tick 结算）：

- 攻方（玩家）优先判定为**胜利**
- 设计意图：避免双方互毒同归于尽的挫败感

### 8.2 超时机制

| 阶段 | 条件 | 效果 |
| -------- | ------------------- | -------------------------------------------- |
| 正常阶段 | cycle_count < 100 | 无额外效果 |
| 加时阶段 | 100 ≤ cycle < 200 | 每周期结算时双方受 `1 + (cycle - 100) × 2` 伤害 |
| 强制结束 | cycle_count ≥ 200 | **超时平局**，双方不获得奖励 |

> **设计意图**：加时伤害从 1 开始递增（1, 3, 5, 7, 9…），增速很快。堆甲 / 堆血的龟缩策略无法无限拖延。实际上绝大多数战斗在加时开始后 10-20 个周期内就会结束。

## 十、战斗 UI 展示建议

### 10.1 行动时间轴可视化

在战斗界面顶部或底部展示一个**时间轴条**，显示双方行动槽的填充进度：

```
[玩家行动槽] ████████░░ 80%     [对手行动槽] ██████░░░░ 60%
```

当某方行动槽满时，播放该角色的出牌动画。玩家可以直观看到"谁即将行动"。

### 10.2 速度变化反馈

当霜蚀 / 汲取导致速度变化时，在角色状态区域显示速度变化动画（如 `SPD 10 → 7 ↓`），同时时间轴条的填充速率发生可见变化，让玩家感受到"对手被减速了"。

### 10.3 战斗日志

侧边可展开的战斗日志，记录每次行动：

```
[Tick 10] 玩家 打出「火焰冲击」→ 对手受到 16 伤害（10 + 灼烧6），灼烧+2
[Tick 15] 对手 打出「铁壁」→ 获得 15 护甲
[Tick 20] 玩家 打出「小毒瓶」→ 对手毒药+5
[Cycle 1] 周期结算：对手毒药伤害 5，霜蚀 12→10，双方 MP+1
```

## 十一、数值参考与平衡锚点

### 11.1 速度区间

| 场景 | 典型速度范围 |
| -------------- | ------------ |
| 开局基础 SPD | 3-15（总点 40 中分配） |
| 刺客加速后 | 15-25 |
| 被冰系控制后 | 0-5 |
| 被血族汲取后 | 视汲取量，可降至 1-3 |

### 11.2 关键平衡参数

| 参数 | 当前值 | 可调范围 | 影响 |
| -------------- | ------ | -------- | -------------------------------- |
| 行动槽阈值 | 100 | 固定 | 基准常量 |
| 霜蚀→减速系数 | 3层/1速 | 2-5 | 越小冰系越强 |
| 霜蚀衰减速率 | 2层/周期 | 1-3 | 越小冰系控制持续越久 |
| 毒药衰减速率 | 1层/周期 | 0-2 | 0 = 不衰减，毒系极强 |
| MP 回复速率 | 1/周期 | 1-2 | 影响高费卡的可用频率 |
| 加时起始周期 | 100 | 50-150 | 赛季特色可调整（如速攻赛季 = 50） |

### 12.3 卡序的战略意义

ATB 系统下卡序更加重要：

- **火系**：叠灼烧牌在前、引爆牌在后
- **守卫**：堆甲牌在前、甲转伤害牌在后
- **武僧**：高蓝条件卡在前（开局满蓝）、回蓝卡在中间、耗蓝爆发卡在后
- **冰系**：霜蚀卡在前，利用减速后的行动优势循环卡组
- **通用**：低费卡和高费卡交错排列，避免连续高费导致 MP 枯竭
