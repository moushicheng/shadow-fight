# shadow-fight

## 一、游戏概述

**类型**：Roguelike 卡牌构筑 + 异步 PVP

**核心卖点**：随机属性、流派构筑、PvE + 异步 PvP——每局随机生成四维属性和可选流派池，通过爬塔中的事件、野怪战斗、残影对决、商店不断构筑卡组。每层先挑战野怪获取资源，再对抗其他玩家留下的"残影"数据。

**参考游戏**：杀戮尖塔（Slay the Spire）

**平台**：抖音小游戏（轻量级，单局 5-15 分钟）

---

## 二、核心循环

> 详细设计见 [`core-loop.md`](core-loop.md)

每层包含 2 个循环，每个循环 3 个节点：

```
┌──────────────────────────── 第 N 层 ────────────────────────────┐
│                                                                  │
│  [事件] → [路线三选一] → [事件] → [事件] → [残影三选一] → [商店] │
│  ╰────── 第 1 循环（PvE） ──╯    ╰────── 第 2 循环（PvP） ──╯  │
│                                                                  │
│                       → 进入第 N+1 层                            │
└──────────────────────────────────────────────────────────────────┘
```

- **10 层制**：1-3 新手 → 4-6 成长 → 7-9 挑战 → 10 Boss
- 死亡存档为残影，通关第 10 层即胜利

## 四、关键设计决策

| 决策 | 选择 | 理由 |
| --- | --- | --- |
| 战斗方式 | 完全自动 | 轻量级小游戏定位，策略在战斗外 |
| 行动系统 | ATB 时间轴 | 速度差异平滑映射，无阶梯效应 |
| 流派池 | 每局随机 2 个 | 45 种组合保证重玩性 |
| HP 管理 | 每场战斗后回满 | 残影按满血计算，不回满则 PvP 不公平 |
| 卡组 | 有序循环 | 卡序编排 = 核心策略深度 |
| PVP 方式 | 异步残影 | 无需等待匹配，适合轻量级 |
