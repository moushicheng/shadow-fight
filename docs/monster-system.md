# 野怪系统

## 一、概述

野怪是每层第 1 循环的 PvE 对手。每个野怪都由开发者**精心设计**——包括属性比例、卡组搭配、携带遗物，而非随机生成。野怪战斗为玩家提供资源积累和卡组测试的机会，难度低于同层残影。

---

## 二、野怪模板设计

每个野怪作为 `MonsterTemplate` 预定义在 `data/monsters.ts` 中，包含以下要素：

| 要素     | 说明                                                                       |
| -------- | -------------------------------------------------------------------------- |
| 基础属性 | 开发者设计的四维比例（STR/CON/SPD/MANA），系统按层数等比缩放到目标总点数   |
| 卡组     | 开发者手工搭配的有序卡组，体现该怪物的战斗风格和策略，不做随机替换         |
| 遗物     | 开发者指定的携带遗物列表，精英怪可预设 1-2 个遗物                          |
| 金币掉落 | 设定 min/max 范围，实际掉落在范围内随机                                    |
| 出现层数 | floorMin ~ floorMax，控制该怪物在哪些层出现                                |

### 属性缩放规则

模板中的属性是**比例模板**，实际战斗时按当前层数缩放：

| 难度     | 属性总点数公式                           | 说明                       |
| -------- | ---------------------------------------- | -------------------------- |
| 普通     | 25 + 层数 × 3                            | 如第 5 层 = 40 点          |
| 精英     | 25 + (层数 + 2) × 3                      | 等效高 2 层的普通难度      |
| 赏金     | round(精英属性总点 × 1.5)                | 精英的 1.5 倍              |

缩放保持模板设计的属性比例（等比缩放 + 舍入修正），SPD 保底 ≥ 1。

## 三、野怪类型

每层从预设的野怪模板池中筛选合适的模板，不同类型怪物有不同的设计目标：

| 类型     | 设计目标                                          | 出现层数 |
| -------- | ------------------------------------------------- | -------- |
| 普通野怪 | 基础对手，教学和资源积累用，卡组简单直接          | 1-10 层  |
| 流派野怪 | 展示某一流派的战斗风格，让玩家体验流派机制        | 3-10 层  |
| 精英野怪 | 高难度挑战，卡组搭配更完善，携带遗物，奖励更好    | 6-10 层  |

### 设计要点

- **普通野怪**：卡组仅含通用（COMMON）卡牌，属性分配均衡，不带遗物
- **流派野怪**：卡组以某一流派卡牌为主、通用卡为辅，属性偏向该流派的核心维度（如冰系偏 SPD）
- **精英野怪**：卡组质量高（可含稀有+卡牌），有明确的战斗策略，携带 1-2 个遗物增加战斗变数

## 四、路线选择中的怪物

路线三选一节点展示每个选项的怪物名称，给玩家决策依据：

| 路线     | 怪物来源          | 属性缩放     |
| -------- | ----------------- | ------------ |
| 普通 A/B | 从当前层合法模板池中各抽一个（不重复） | 普通难度     |
| 精英     | 优先选 ELITE 类型模板，无则退化为任意  | 精英难度     |
| 赏金     | 10% 概率出现，优先选 ELITE 模板        | 赏金难度     |

1-2 层路线锁定为普通战斗（无三选一），3 层起解锁选择。

---

## 五、野怪与残影的对比

| 维度     | 野怪（第 1 循环）             | 残影（第 2 循环）                  |
| -------- | ----------------------------- | ---------------------------------- |
| 来源     | 开发者预设模板 + 属性缩放     | 其他玩家的死亡存档                 |
| 难度     | 较低，侧重资源积累            | 较高，考验构筑能力                 |
| 社交属性 | 无                            | 显示对手昵称/头像，支持复仇        |
| 奖励     | 基础金币 + 卡牌选择           | 更高金币 + 卡牌选择 + 更高遗物概率 |
| 战后节点 | 进入第 2 循环事件             | 进入商店                           |

> **策划意图**：野怪作为 PvE 缓冲层，降低了每层的整体挫败感。玩家即使在残影战斗中失败，至少在野怪战斗中获得了一些资源和成长。同时野怪战斗也是新手熟悉机制的安全空间。

---

## 六、模板设计规范

开发者设计新野怪时遵循以下规范：

```typescript
{
    id: 'ice_wolf',           // 唯一ID：{流派}_{蛇形命名}
    name: '冰原狼',           // 显示名称
    type: MonsterType.FACTION, // NORMAL / FACTION / ELITE
    faction: Faction.ICE,      // 流派野怪必填，普通野怪不填
    floorMin: 3,               // 最低出现层
    floorMax: 8,               // 最高出现层
    baseProperty: {            // 属性比例模板（会被等比缩放）
        STR: 4, CON: 5, SPD: 8, MANA: 3
    },
    deck: [                    // 精心设计的有序卡组
        { defId: 'ice_frost_bolt', upgraded: false },
        { defId: 'ice_chill', upgraded: false },
        { defId: 'common_guard', upgraded: false },
    ],
    relics: [],                // 携带遗物ID列表
    goldDrop: { min: 15, max: 25 },
}
```

### 每层建议模板数量

| 层数范围 | 建议模板数 | 组成                                     |
| -------- | ---------- | ---------------------------------------- |
| 1-2 层   | 3-5 个     | 全部 NORMAL                              |
| 3-5 层   | 5-8 个     | 3-4 NORMAL + 2-4 FACTION                 |
| 6-9 层   | 6-10 个    | 2-3 NORMAL + 3-4 FACTION + 1-3 ELITE    |
| 10 层    | 2-3 个     | Boss 级 ELITE（由残影系统处理）           |
