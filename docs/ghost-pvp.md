# 异步 PVP — 残影系统

## 一、核心机制

- 玩家死亡时，系统记录其**死亡层数 + 当前卡组 + 基础属性 + 遗物列表**
- 该数据作为「残影」存入对应层的**对手池**
- 残影**仅出现在每层的第 2 循环（残影战斗节点）**，第 1 循环为野怪战斗
- 其他玩家打到该层的第 2 循环时，从对手池中随机匹配残影作为对手
- 残影的战斗逻辑与真人完全一致（按卡序自动出牌）

---

## 二、残影数据结构

```typescript
type Ghost = {
  oderId: string;
  playerName: string;
  seed: number; // 随机种子（用于重建属性）
  baseProperty: {
    // 死亡时的基础属性（含事件修改）
    STR: number;
    CON: number;
    SPD: number;
    MANA: number;
  };
  deck: string[]; // 有序卡组
  relics: string[]; // 遗物列表
  floor: number; // 死在第几层
  factionPool: string[]; // 流派池
  timestamp: number;
};
```

## 三、对手池与三选一匹配规则

残影战斗采用**三选一**机制（详见 [core-loop.md](core-loop.md) 第四节），从对手池中抽出 3 个残影供玩家选择。

| 规则     | 说明                                                              |
| -------- | ----------------------------------------------------------------- |
| 抽取数量 | 从同层残影池中抽取 **3 个**残影，展示概要后由玩家选择 1 个挑战    |
| 难度分布 | 3 个残影尽量覆盖不同难度星级（1 弱 + 1 中 + 1 强），避免三个同档 |
| HP 规则  | 残影 HP = maxHp × 层数难度系数（不用死亡时 HP，避免太弱）        |
| 优先展示 | 复仇名单、好友残影在 3 个候选位中优先占位                        |
| 冷启动   | 残影不足 3 个时使用预设 AI 补齐                                   |
| 数据过期 | 残影保留**当前赛季**内有效                                        |
| 去重     | 同一局不会遇到同一残影两次                                        |
